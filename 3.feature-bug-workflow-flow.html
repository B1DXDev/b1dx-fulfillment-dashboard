START
🐛 Bug Report
USER: เจอ bug: {symptom}
PHASE 0 — TRIAGE
📋 บันทึก backlog § Issues

🔍 Reproduce + Root Cause

📝 plan.md — symptom · root cause · fix · regression test
USER: build / ลุย ⭐
PHASE 2 — TDD (BLOCKING)
🧪 เขียน Regression Test (failing ก่อน)
test ที่ reproduce bug → รัน → ต้อง FAIL ก่อน
🔧 Fix Code
แก้ root cause — minimal change
PHASE 3 — VERIFY
✅ Regression Test PASS + Suite เดิมยังผ่าน
⚠️ ห้าม auto commit — รอ user สั่ง

USER: push
⬆️ git push origin fix/{branch}
USER: PR
🔀 gh pr create + repro steps + screenshot
USER: merged: {BUG-ID}

🎉 backlog § Issues → ✓ Resolved

2A
เขียน Failing Regression Test ก่อน TDD BLOCKING
test ก่อนแก้โค้ด — ขั้นที่ต่างจาก feature ชัดที่สุด
2A.1
Agent เขียน test ที่ reproduce bug
AGENT ทำ BE: unit/integration test · FE: E2E test (in e2e/{bug-id}/)
test name: regression_{bug-id}_{scenario}
2A.2
รัน test — ต้อง FAIL
AGENT ทำ dotnet test / pnpm test / playwright test
ถ้า test ผ่านตั้งแต่ยังไม่แก้ = test เขียนผิด ต้องเขียนใหม่ให้ fail ก่อน
▼
2B
Fix Code AUTO-PROCEED
แก้ root cause — minimal change ไม่เพิ่ม scope
2B.1
Agent แก้โค้ดที่ root cause
AGENT ทำ ① git checkout main && pull && checkout -b fix/{bug-id}
② แก้เฉพาะที่จำเป็น ห้ามทำ refactor นอก scope
③ ถ้าเจอ bug อื่น → log ใน backlog.md แล้วถาม user ก่อนแก้ (Stop List #11)
▼
3
Verify + Stage
regression test ผ่าน + suite เดิมยังผ่าน
3.1
Regression test PASS
AGENT ทำ test ที่เคย fail ตอนนี้ต้องผ่าน — ถ้ายัง fail = fix ไม่ตรง root cause
3.2
Test suite เดิมยังผ่านทั้งหมด
AGENT ทำ ห้ามมี test เดิม fail หลัง fix — ถ้ามีต้องหยุดถาม user (Stop List #12)
3.3
Stage + Report
AGENT ทำ git add + รายงาน: branch · ไฟล์ · commit msg ที่ร่าง (fix: ...) · test result
USER ต่อไป ตัดสินใจสั่ง commit / push / PR

🆚 Bug Fix vs New Feature — ต่างกันตรงไหน

ขั้น🚀 Feature🐛 Bug Fix
Trigger สร้าง feature X เจอ bug: ...
Phase 1 Design + Spec + Mockup Reproduce + Root Cause
Phase 1.5 Scaffold E2E (test.skip) —
Phase 2 — Test เขียน test คู่กับ impl เขียน failing regression test ก่อน fix
Phase 2 — Code Build feature ใหม่ Minimal fix ที่ root cause
Branch feat/{name} fix/{bug-id}
Commit prefix feat: fix:
PR ต้องมี Screenshot · Mockup fidelity Repro · Root cause · Before/After screenshot
Done update plan.md + sprint backlog § Issues → ✓ Resolved + plan.md + sprint
⚡ TLDR — จุดสำคัญที่ต้องจำ
B1DX Feature Workflow — สรุปก่อนดู flow ยาว
🟡 3 จุดที่ User ต้อง "สั่ง" GATE
  • หลัง Design → confirmed / approved
  • หลัง Plan Gate → build / ลุย ⭐
    trigger หลัก · Status: Draft → Confirmed · agent รันยาวไม่หยุด
  • หลัง Verify → commit → push → PR → สุดท้าย merged: {TICKET}
🟢 ขั้นที่ Agent ทำเองยาวๆ AUTO
  • Phase 1.5 Scaffold E2E tests (test.skip)
  • Phase 2 Build parallel — 4 subagent (BE / FE / Sync / Webhook)
  • ⚠️ Auto-proceed ตลอด ยกเว้นชน Stop List 13 เงื่อนไข (scope change, breaking change, irreversible ฯลฯ)
🔴 ข้อระวัง RULES
  • Phase 3 verify เสร็จ → agent git add ได้
    แต่ห้าม auto commit/push/PR — กฎเด็ดขาด
  • ห้าม push main ตรง → ต้อง yes push main explicit เท่านั้น
  • UI changes ต้อง verify visual fidelity vs mockup ก่อน mark done (ไม่ใช่แค่ test pass)
