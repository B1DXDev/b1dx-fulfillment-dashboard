📋 Workflow Summary — b1dx-fulfillment-workspace

สรุป workflow, lifecycle, และ commands ของ solution นี้

1. Workflow Diagram

🎨 PHASE 1 — Design

  1. อ่าน be-repo/CLAUDE.md + web-repo/CLAUDE.md
  2. อ่าน mockup ที่ spec ระบุ (fallback: docs/mockups/b1dx_full_app_v0.html)
  3. ออกแบบ data model + API contract
  4. เขียน spec → docs/specs/{feature}.md
  5. แสดง test plan + รับ confirm จาก user
▼

🚦 Pre-Build Confirmation Gate (บังคับ)

  • Step 1 — present 4 รายการ: Plan & Scope / Summary of changes / SIT Impact / Test Plan
  • Step 2 — สร้าง tasks/plans/{TICKET-ID}/plan.md (Status: Draft)
  • Step 3 — รอ user พิมพ์ "confirm" / "go" / "ยืนยัน" → Status: Confirmed
▼

🧪 PHASE 1.5 — Scaffold Tests

UI Tester subagent → e2e/{feature}/{feature}.spec.ts (test.skip ทุกตัว)

▼

⚙️ PHASE 2 — Build (parallel)

BE Subagent

Domain entity+test → Application command/query+test → Infrastructure migration → Api endpoint → integration test

dotnet build && dotnet test

FE Subagent

types → zod schema → MSW mock → hook+test → component+test → page

pnpm lint && pnpm typecheck && pnpm test

Sync Worker (ถ้าเกี่ยว sync)

Domain entity+test → Application service+test → Infrastructure → BackgroundJob

dotnet build && dotnet test

ทุกตัวต้อง: git status --short → git checkout main && git pull && git checkout -b {branch} → TDD → push → node scripts/gen-progress.mjs

▼

✅ PHASE 3 — Verify & Report

  • UI Tester un-skip + run playwright test e2e/{feature}/ → ผ่านทั้งหมด
  • รัน node scripts/gen-progress.mjs
  • รายงาน user: ไฟล์ที่สร้าง/แก้ + test result

🚀 Pipeline / Build

claude "handoff S2-BE-01 from nuchit to wat"
claude "suggest parallel task: role=fe — waiting for S2-BE-01"

# dev ใหม่ / เดือนใหม่
cp tasks/assignments/_template.md tasks/assignments/$(date +%Y-%m)-{name}.md
