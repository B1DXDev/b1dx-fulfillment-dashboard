Multi-agent pipeline แบบ Agent Team Orchestrator Pattern สำหรับ b1dx-fulfillment-workspace — orchestration repo คุม 4 sub-repos (BE / FE / Sync Worker / Webhook) พร้อม lifecycle, agent roles และ commands

1Pipeline Diagram

End-to-end flow: Design → Plan Gate → Scaffold Tests → Build (parallel) → Verify & Report

2Phase Details

🎨

Phase 1 — Design

Architect + Spec Writer
  1. อ่าน be-repo/CLAUDE.md + web-repo/CLAUDE.md
  2. อ่าน mockup ที่ spec ระบุ (fallback: docs/mockups/b1dx_full_app_v0.html)
  3. ออกแบบ data model + API contract
  4. เขียน spec → docs/specs/{feature}.md
  5. แสดง test plan + รับ confirm จาก user ก่อนเข้า Plan Gate
🚦

Pre-Build Plan Gate

BLOCKING · 1 gate / task
  • Step 1 — present 4 รายการใน chat: Plan & Scope / Summary of changes / SIT Impact / Test Plan
  • Step 2 — สร้าง tasks/plans/{TICKET-ID}/plan.md (Status: Draft) + ปิดท้ายด้วย call-to-action เช่น "พิมพ์ build หรือ ลุย เพื่อเริ่ม"
  • Step 3 — รอ user trigger: primary build / ลุย, secondary confirm / go / ยืนยัน → Status: Confirmed → build ต่อเนื่อง ไม่ pause ยกเว้นชน Auto-Proceed Stop List (13 ข้อ)
🧪

Phase 1.5 — Scaffold Tests

UI Tester

spawn UI Tester subagent → สร้าง e2e/{feature}/{feature}.spec.ts (test.skip ทุกตัว) ตาม acceptance criteria; feature = happy + edge; bug = regression reproduce; improve = before/after

commit: test(e2e): scaffold {feature} acceptance tests

⚙️

Phase 2 — Build (Parallel)

TDD · 4 subagents

BE

.NET 10

Domain entity+test → Application command/query+test → Infrastructure migration → Api endpoint → integration test

dotnet build && dotnet test

FE

Next.js 15

types → zod schema → MSW mock → hook+test → component+test → page

pnpm lint && pnpm typecheck && pnpm test

Sync Worker

.NET 10

Domain entity+test → Application service+test → Infrastructure → BackgroundJob

dotnet build && dotnet test

Webhook

Go 1.25

Domain → Service → Adapter (platform-specific) → Handler (HTTP 202 + RabbitMQ publish) → Worker consumer → integration

go build && go test ./...

ทุกตัวต้อง: git status --short → git checkout main && git pull → git checkout -b {branch} → TDD → push → node scripts/gen-progress.mjs

✅

Phase 3 — Verify & Report

Test pass = required
  • UI Tester un-skip + run playwright test e2e/{feature}/ → ผ่านทั้งหมด
  • รัน node scripts/gen-progress.mjs ที่ workspace root
  • รายงาน user: ไฟล์ที่สร้าง/แก้ + test result + screenshot (ถ้ามี UI change)
  • Fail → กลับไปแก้ที่ Phase 2 ก่อน ห้าม report เสร็จ

3Lifecycle

Task Lifecycle

in tasks/assignments/YYYY-MM-{name}.md

[ ] Todo → [~] Active → [👀] In Review → [x] Done

Plan Status

in tasks/plans/{TICKET-ID}/plan.md

Draft → Confirmed → In Progress → In Review → Done
⚠️ 3 ไฟล์ต้อง sync พร้อมกันเมื่อ assign task
  1. tasks/backlog.md § Issues Backlog — ID, symptom, root cause, test cases
  2. tasks/current-sprint.md — effort, files, scope, AC, branch
  3. tasks/assignments/YYYY-MM-{name}.md § Active — branch + claimed date
