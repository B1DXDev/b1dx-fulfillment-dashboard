Why Workspace + Agent Team?

1 flow · 1 diagram · 6 ข้อดีหลัก — ทำไม B1dx ใช้ workspace กลาง + 9 agent แทนการให้แต่ละ repo ทำงานแยกกัน

01The Flow — Lifecycle ของ 1 Task

จาก "อยากได้ feature นี้" → "merged แล้ว" ผ่าน gate ไหนบ้าง · ทุก step มี benefit ติดอยู่ข้างๆ

หัวใจของ flow นี้ ทุก gate (🛡️ Plan, 🎨 Visual) คือ มนุษย์ตรวจ agent ก่อนเข้า step ถัดไป — agent ทำงานเร็ว แต่ตัดสินใจไม่ได้ · gate ทำให้ความเร็วของ agent ไม่กลายเป็นความผิดพลาดเร็ว

02The Diagram — Workspace Ecosystem

ภาพรวม "ใครอยู่ตรงไหน อ่านอะไร เขียนที่ไหน" — ทุกอย่างหมุนรอบ Workspace กลาง ที่เป็น single source of truth

Key insight Workspace = "ตัวกลาง" ที่ทำให้ทุกคน (agent + คน + repo) มอง เป้าหมายเดียวกัน · ไม่ใช่แค่ folder ที่เก็บไฟล์ — มันคือ "ที่นัดเจอ" ของทีม

036 ข้อดีที่ได้จริง

📋

Task Tracking

เห็นใครทำอะไรจาก assignments/ + plans/ ไม่ต้องถามใคร

🎯

Single Source of Truth

spec + mockup + CLAUDE.md ที่เดียว — แก้ครั้งเดียวทุกฝั่งเห็น

⚡

Parallel Build

BE + FE + Sync + Webhook ทำพร้อมกัน · FE ไม่ต้องรอ BE (ใช้ MSW)

🛡️

Quality Gates

Plan Gate + Visual Fidelity + Test Gate — กัน spec drift

🚀

Fast Onboarding

Dev ใหม่ < 30 นาที (clone + config 4 paths + claude)

💰

Token Efficient

RTK proxy + subagent delegation = ประหยัด 60-90%

Bottom line Workspace + Agent Team ไม่ใช่ "เครื่องมือทันสมัย" — มันคือ infrastructure ของความสอดคล้อง ที่ทำให้ทีมเล็กส่งงานคุณภาพระดับทีมใหญ่ และทำให้ทีมใหญ่ไม่จมด้วย coordination overhead
