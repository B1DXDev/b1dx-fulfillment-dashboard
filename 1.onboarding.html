01Overview

โปรเจกต์นี้ใช้ workspace pattern — มี repo กลาง 1 ตัวที่รวม 4 repos (BE / FE / Sync Worker / Webhook) ไว้เป็น git submodule เพื่อให้ Claude Code อ่าน spec, mockup, convention ได้จากที่เดียว

b1dx-fulfillment-workspace/      ← clone ที่นี่
├── be-repo/                     ← submodule (read spec/CLAUDE.md เท่านั้น)
├── web-repo/                    ← submodule (read spec/CLAUDE.md เท่านั้น)
├── sync-worker-repo/            ← submodule (read spec/CLAUDE.md เท่านั้น)
├── webhook-repo/                ← submodule (read spec/CLAUDE.md เท่านั้น)
├── agents/                      ← system prompts ของ subagent ทุกตัว
├── docs/{specs,mockups,guides,adr,standup,runbooks}/
├── tasks/{current-sprint,backlog}.md + {assignments,plans,sprints,archive}/
├── scripts/                     ← gen-progress, weekly, standup, sync:gh
├── workspace.local.json         ← config path จริง (gitignored — ต้องสร้างเอง)
└── workspace.local.example.json ← template

Repos จริงที่เขียนโค้ด clone แยกต่างหากในเครื่องของแต่ละคน — submodule ใน workspace เป็น read-only spec mirror เท่านั้น:

LayerRepoStack
BEb1dx-oms-fulfillment-api.NET 10, EF Core, PostgreSQL, Mediator
FEb1dx-oms-fulfillment-webNext.js 15, RTK Query, MSW, Playwright
Sync Workerb1dx.omsorders.sync.worker.NET 10 Worker (ERP→OMS sync)
WebhookwebhookGo 1.25, Gin, GORM, RabbitMQ

03Clone repos

# 1. Clone workspace (พร้อม submodule ทุกตัว)
git clone --recurse-submodules https://github.com/B1DXDev/b1dx-fulfillment-workspace.git

# 2. Clone repos จริง (ไว้นอก workspace ที่ไหนก็ได้)
git clone https://github.com/B1DXDev/b1dx-oms-fulfillment-api.git
git clone https://github.com/B1DXDev/b1dx-oms-fulfillment-web.git
git clone https://github.com/B1DXDev/b1dx.omsorders.sync.worker.git    # ถ้าทำงาน Sync
git clone https://github.com/B1DXDev/webhook.git                       # ถ้าทำงาน Webhook

วาง repos ที่ไหนก็ได้ — path จะ config ในขั้นต่อไป

💡
Sync Worker / Webhook clone เฉพาะตอนต้องทำ feature ที่เกี่ยวข้องก็พอ — ไม่จำเป็นต้อง clone ทุกตัวตั้งแต่วันแรก

04ตั้งค่า workspace.local.json

cd b1dx-fulfillment-workspace
cp workspace.local.example.json workspace.local.json

เปิด workspace.local.json แก้ path ให้ตรงกับเครื่องของตัวเอง:

{
  "be_repo_path": "/Users/yourname/projects/b1dx-oms-fulfillment-api",
  "fe_repo_path": "/Users/yourname/projects/b1dx-oms-fulfillment-web",
  "sync_worker_repo_path": "/Users/yourname/projects/b1dx.omsorders.sync.worker",
  "webhook_repo_path": "/Users/yourname/projects/webhook"
}
⚠️
ไฟล์นี้อยู่ใน .gitignore ไม่ถูก commit ไม่ต้อง push — ถ้ายังไม่ได้ clone Sync Worker / Webhook ใส่ path ว่างไว้ก่อนได้ (ใช้ตอนต้อง spawn subagent ตัวนั้นเท่านั้น)

05ตั้งค่า FE (.env.local)

cd {fe_repo_path}/apps/b1dx-oms-fulfillment
cp .env.example .env.local

ค่า default สำหรับ local dev:

NEXT_PUBLIC_MSW=1          # ใช้ mock data (ไม่ต้องรัน BE)
CORE_API_BASE_URL=https://localhost:7143
GATEWAY_ALLOW_PRIVATE_HOSTS=1

เปลี่ยนเป็น NEXT_PUBLIC_MSW=0 เมื่อต้องการเชื่อมกับ BE จริง

07ใช้ Claude Code

เปิด Claude Code จาก workspace root เสมอ:

cd b1dx-fulfillment-workspace
claude

Claude จะอ่าน CLAUDE.md ที่นี่เป็นอันดับแรก และรู้ว่า BE/FE/Sync/Webhook อยู่ที่ path ใดจาก workspace.local.json

คำสั่งที่ใช้บ่อย

# Full pipeline (Orchestrator spawn ครบทุก agent)
claude "อ่าน agents/orchestrator.md + tasks/current-sprint.md แล้ว build ตาม Working Protocol ทุก Phase"

# Build feature ใหม่
claude "อ่าน agents/orchestrator.md แล้ว build feature {ชื่อ}: {อธิบาย}"

# ฝั่งเดียว
claude "build BE for {feature} — อ่าน be-repo/CLAUDE.md"
claude "build FE for {feature} — อ่าน docs/specs/{feature}.md และ web-repo/CLAUDE.md"

# Task Assignment
claude "assign next task: role=be"             # หรือ role=fe / fullstack
claude "pr: S2-BE-01 — {pr-url}"               # → In Review
claude "merged: S2-BE-01"                        # → Done

# Sprint Planning
claude "อ่าน agents/sprint-planner.md แล้ว plan sprint ถัดไป — capacity: {X} วัน, {N} dev"

npm scripts (จาก workspace root)

npm run standup           # Daily standup (auto-detect owner)
npm run weekly            # Weekly report (ทุกศุกร์)
npm run progress          # Generate progress dashboard
npm run progress:serve    # http://localhost:3939
npm run progress:watch    # Auto-regenerate
npm run sync:gh:dry       # Dry-run sync tasks ↔ GitHub Issues

Pre-Build Plan Gate (สำคัญ)

ทุก task ก่อนแก้โค้ดแม้แต่บรรทัดเดียว Claude จะ:

  1. แสดง Plan & Scope / Summary of changes / SIT Impact / Test Plan ใน chat
  2. สร้าง tasks/plans/{TICKET-ID}/plan.md (Status: Draft)
  3. รอ user trigger: พิมพ์ build หรือ ลุย (หรือ confirm / go / ยืนยัน) → Claude เริ่ม build ต่อเนื่อง ไม่ confirm กลางทาง ยกเว้นชน Auto-Proceed Stop List
📖
รายละเอียดเต็มของ Auto-Proceed Stop List (13 เงื่อนไข) อ่านที่ CLAUDE.md § Auto-Proceed Stop List

08Git workflow

🚨
ห้าม push ตรงเข้า main BLOCKING — hardened 2026-05-13 หลัง violation 3 ครั้งวันเดียว · feature ต้อง PR + reviewer merge เสมอ · Agent ถ้าเห็น sequence checkout main && merge feat/x && push origin main ต้อง hard-stop + ขอ confirm yes push main ทุกครั้ง 
# ใน working dir จริง (ไม่ใช่ submodule)
cd {be_repo_path}  # หรือ {fe_repo_path} / {sync_worker_repo_path} / {webhook_repo_path}

git status --short                         # ตรวจ local changes ก่อนเสมอ
git checkout main && git pull origin main
git checkout -b feat/your-feature

# commit & push
git add src/ && git commit -m "feat: ..."
git push -u origin feat/your-feature

# PR ผ่าน gh CLI
gh pr create --title "..." --body "..."

# หลัง push: regen progress dashboard ที่ workspace root
cd {workspace_root} && node scripts/gen-progress.mjs
✋
Commit/Push Boundary Agent stage ไฟล์ได้ (git add) แต่ ห้าม auto commit / push / gh pr create หลัง edit เสร็จ — ต้องหยุดที่ stage + รอ user สั่งก่อน

09กฎสำคัญ — ห้ามลืม

กฎเหตุผล
ห้ามเขียนโค้ดลง submodule (./be-repo/, ./web-repo/, ./sync-worker-repo/, ./webhook-repo/)submodule ใช้สำหรับอ่าน spec/CLAUDE.md เท่านั้น — เขียนใน working dir จริง
รัน git status --short ก่อน build ทุกครั้งป้องกัน Claude เขียนทับงานที่ทำค้างไว้
แตก branch ก่อนเสมอ + ห้าม push ตรงเข้า mainBLOCKING — ผ่าน PR + reviewer merge เท่านั้น
Bug/Issue ใหม่ → tasks/backlog.md § Issues Backlogห้ามสร้างไฟล์ใหม่ใน docs/issues/ (deprecated)
Spec อยู่ที่ docs/specs/{feature}.md เท่านั้นไฟล์เดียว BE+FE ใช้ร่วม — ห้าม copy
Mockup Visual Fidelity = acceptance criteriatests passing ≠ feature เสร็จ — ต้องเทียบ screenshot vs mockup ก่อน mark done
Test = blocking (ทุก task)FE: E2E scaffold ก่อน impl · BE: unit + integration · ไม่มี test = ห้าม report เสร็จ
Task Assignment ต้องอัพเดท 3 ไฟล์พร้อมกันbacklog.md + current-sprint.md + assignments/YYYY-MM-{name}.md
หลัง push ทุกครั้ง: node scripts/gen-progress.mjsไม่มี pre-commit hook auto-regen แล้ว
BE: ห้ามรับ tenant_id จาก request bodyใช้ ICurrentTenant เสมอ (tenant isolation)
FE: ห้าม <a href> ใน (app) routesใช้ router.push() เสมอ (Next.js SPA navigation)
FE: ห้าม deep import เช่น @b1dx/ui/src/components/ui/buttonใช้ public entry ของ package เท่านั้น
New PostgreSQL tables/columns ใช้ snake_caseห้าม PascalCase (project convention)
อย่า push workspace.local.jsongitignored — มี path เฉพาะเครื่องตัวเอง

13Daily Workflow (สำหรับ dev)

# เช้า — claim task ของวันนี้
claude "assign next task: role=be"             # หรือ fe / fullstack

# ระหว่างวัน — build feature (Claude จะแสดง plan → รอ user trigger)
claude "อ่าน agents/orchestrator.md แล้ว build {TICKET-ID}"
# ↑ พิมพ์ `build` หรือ `ลุย` เพื่อ trigger หลังเห็น plan

# หลัง push PR
claude "pr: S2-BE-01 — https://github.com/.../pull/42"
node scripts/gen-progress.mjs

# เย็น — daily standup
npm run standup
🔰
ครั้งแรก: สร้าง assignment file ของตัวเอง cp tasks/assignments/_template.md tasks/assignments/$(date +%Y-%m)-{yourname}.md

14Troubleshooting

Submodule ไม่อัปเดต

git submodule update --remote --merge

Submodule HEAD ชี้ผิด branch

git -C be-repo checkout main && git -C be-repo pull origin main
git -C web-repo checkout main && git -C web-repo pull origin main

FE typecheck fail

cd {fe_repo_path}
pnpm --filter @b1dx/ui build   # build UI package ก่อน
pnpm typecheck

BE build fail

cd {be_repo_path}
dotnet restore
dotnet build
