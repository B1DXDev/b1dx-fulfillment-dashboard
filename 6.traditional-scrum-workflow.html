Dev เปลี่ยนจาก “คนเขียนโค้ด” เป็น “คนคุม AI”
บทบาทใหม่ที่สำคัญกว่าเดิม — เพราะ AI ตัดสินใจแทนไม่ได้

บทบาทใหม่ของ Dev (6 หน้าที่หลัก)

1. Architect
  • ออกแบบ system architecture
  • ตัดสินใจเลือก tech stack
  • วาง pattern ให้ AI ทำตาม
2. Prompt Engineer
  • เขียน prompt/spec ให้ชัดเจน
  • แตก task ให้ AI ทำได้
  • กำหนด context ที่ต้องใช้
3. Code Reviewer
  • ตรวจ PR ที่ AI สร้าง
  • หา bug, security issue
  • ยืนยันว่าตรงตาม business
4. Debugger / Problem Solver
  • แก้ปัญหาที่ AI แก้ไม่ได้
  • เทส edge case ยับ ๆ
  • Production incident
5. Domain Expert
  • เข้าใจ business จริง ๆ
  • คุยกับ user / stakeholder
  • แปล req เป็น tech solution
6. Quality Gatekeeper
  • รับผิดชอบ production
  • ตัดสินใจ deploy / rollback
  • ดูแล performance, cost

หัวใจสำคัญ: Dev ไม่ได้หายไป แต่ “ยกระดับขึ้น”
  • AI ทำงาน routine ได้ดี แต่ “ตัดสินใจ” ไม่ได้ → Dev ต้องเป็นคนตัดสินใจ
  • AI ไม่เข้าใจ business context ลึก ๆ → Dev ต้องเป็นสะพานเชื่อม
  • AI สร้าง bug แต่ไม่เหมือนคน → Dev ต้องเป็นด่านสุดท้ายก่อน production
  • AI ไม่รับผิดชอบเมื่อระบบล่ม → Dev ยังต้อง on-call เสมอ
  • Dev junior → ต้องเรียนรู้เร็วขึ้น เพื่อ review งาน AI ได้
  • Dev senior → คุณค่าสูงขึ้น เพราะประสบการณ์ตัดสินใจหายาก
