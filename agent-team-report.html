⚖️ Single Agent vs Agent Team

❌ Single Agent (แบบเก่า)

  • 1 context ทำทุกอย่างเอง
  • System prompt เดียว ครอบคลุมทุกเรื่อง
  • ทำงานแบบ sequential ทีละขั้น
  • Context พองเร็ว — สูญเสีย accuracy
  • ไม่มี separation of concerns
  • ไม่มี quality gate ที่ enforce ได้

✅ Agent Team (Solution นี้)

  • Orchestrator + spawn specialist subagents
  • 9 system prompts เฉพาะแต่ละ role
  • Parallel — BE+FE+Sync ทำพร้อมกัน
  • Context แต่ละ subagent แยก ไม่ปนกัน
  • BE / FE / Sync / Test / Review แยกชัด
  • Reviewer + UI Tester เป็น quality gate

🎁 Business Benefits

⚡ Speed
BE + FE + Sync Worker ทำงาน parallel — ลดเวลา delivery feature 2-3 เท่า เทียบกับ sequential
🎯 Quality
UI Tester + Reviewer เป็น mandatory gate — ป้องกัน visual regression และ code quality issue
📊 Traceability
ทุก task มี plan.md + lifecycle status (Todo→Done) — ติดตามได้ทุกขั้นตอน
🔒 Consistency
แต่ละ specialist มี CLAUDE.md + system prompt — บังคับ pattern เดียวกันทุกครั้ง
💰 Cost Efficiency
Context isolation + RTK token optimizer — ประหยัด token 60-90% ต่อ operation
📈 Scalability
เพิ่ม agent ใหม่ได้ง่าย (เช่น Mobile Coder, DBA) โดยไม่กระทบ agent เดิม

💬 รูปแบบการสื่อสาร — Hierarchical (ไม่ใช่ Peer-to-Peer)

รูปแบบSolution นี้หมายเหตุ
Hierarchical (Coordinator → Workers) ✓ ใช้ Orchestrator spawn subagent → return ผลกลับ
Peer-to-Peer (Agents คุยกันเอง) ✗ ไม่ใช้ ไม่มี agent คุยกันโดยตรง
Shared File State ✓ ใช้ spec.md, plan.md, backlog.md, assignments/
Context Isolation ✓ ใช้ subagent return summary ไม่ pollute main context
