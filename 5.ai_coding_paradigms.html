Evolution Timeline · จาก AI Auto Complete สู่ AI Software Organization
6 ยุคของ AI ใน software development · ตัวอย่าง · ความสามารถ · บทบาทของ Dev ในแต่ละยุค
2018
— 2018–2020 · ยุค AI Auto Complete

AI Auto Complete Era

AI เริ่มช่วยเติม code ระดับ token/บรรทัด ผ่าน GitHub Copilot Prototype, IntelliSense++, Tab Completion ทำงานตามไวยากรณ์ ไม่มี context ทั้ง project ไม่เข้าใจ architecture หรือ business

ตัวอย่าง
GitHub Copilot Prototype · IntelliSense++ · Tab Completion · ctags
AI ทำได้
เติม code · constructor · property · loop · boilerplate · syntax help
Dev อยู่ตรงไหน
Driver 95% · AI 5%นักพัฒนาเขียนทั้งหมด ไม่มี context ไม่เข้า architecture
2021
— 2021–2023 · ยุค AI Assistant

AI Assistant Era

ChatGPT และ Copilot Chat เปลี่ยน AI เป็นผู้ช่วยที่เข้าใจ conversation, requirement และอธิบาย architecture ได้ — ยุคแรกของ Prompt Engineering และ Spec → Code workflow

ตัวอย่าง
OpenAI ChatGPT · GitHub Copilot Chat · Bard · Claude 1/2
AI ทำได้
generate function · explain code · debug · refactor · generate SQL/API · spec → code
Dev อยู่ตรงไหน
Architect + ReviewerAI = Junior Developer · ยุคแรกของ Prompt Engineering
2023
— 2023–2024 · ยุค AI Pair Programmer · context-centric

AI Pair Programmer Era

จุดเปลี่ยน — AI เริ่ม “เห็นทั้ง project” อ่านหลายไฟล์ เข้าใจ dependency refactor ข้าม module ได้ โลกขยับจาก code-centric → context-centric ต้องมี CLAUDE.md, ARCHITECTURE.md, repo rules

ตัวอย่าง
Cursor · Continue · Codeium · multi-file context · @-mention · shared rules
AI ทำได้
อ่านหลายไฟล์ · เข้าใจ dependency · refactor ข้าม module · generate test · analyze architecture
Dev อยู่ตรงไหน
Tech LeadAI = Mid/Senior Developer · จัดการ context + coding standards
2024
— 2024–2025 · ยุค AI Agent · Autonomous Task Execution

AI Agent Era

AI ไม่ได้แค่ “ตอบ” แต่ “ลงมือทำ” — มี memory, tools, planning, retry รัน task end-to-end อ่าน spec แก้ code รัน test เปิด PR deploy ได้ ยุคของ OpenSpec, Spec-driven Development และ AI Workflow

ตัวอย่าง
Claude Code · Cursor Agent · Devin · OpenSpec · MCP · agentic loop · tool use
AI ทำได้
อ่าน spec · แก้ code · run test · เปิด PR · deploy · workflow · memory + planning + retry
Dev อยู่ตรงไหน
Product/Architecture OwnerAI = Autonomous Engineer · ระวัง context drift · token cost

— 2025–2026 · ยุค Multi-Agent · ปัจจุบัน

Multi-Agent / Agent Team Era

AI ตัวเดียว “ไม่พอ” เกิด Orchestrator + Subagents + Reviewer + Specialist Agents ทำงานขนานเหมือนทีมจริง AI บริหารงาน software engineering ทั้งระบบ — parallel execution, PR review, conflict resolution, workflow orchestration

ตัวอย่าง

Security as a Cross-cutting Concern
4 phases · shift-left security · ความปลอดภัยฝังในทุกขั้นตอน
PHASE 01
ขั้นวิเคราะห์
Security ทำงานคู่กับ BA เพื่อทำ threat modeling ระบุข้อมูลละเอียดอ่อน เส้นทางโจมตี และความเสี่ยงตั้งแต่ก่อนเขียนโค้ดบรรทัดแรก
PHASE 02
ขั้นพัฒนา
Security ตรวจสอบโค้ดจาก Backend และ Frontend เน้น input validation, authentication, secret management, XSS, CSRF, SQL injection
PHASE 03
ขั้นทดสอบ
Security ออกแบบ test case ที่มุมโจมตีร่วมกับ QA และ Tester เช่น fuzzing, penetration test, OWASP ZAP scan ในขั้น E2E
PHASE 04
ขั้นส่งมอบ
Security ตรวจ DevOps pipeline สแกน container image, dependency vulnerability, IaC misconfig และตั้งค่า cloud ที่ปลอดภัย
