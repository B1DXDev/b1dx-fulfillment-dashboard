Section 02
Register Client — ทำตามลำดับ (กดติ๊กเก็บ progress ได้)
0 / 6 เสร็จ
  1. 01Login Admin Console
    เปิด https://keycloak.benzapps.com/admin/ → login ด้วย account ที่มีสิทธิ์ realm admin → เลือก realm b1dx ที่มุมซ้ายบน (ไม่ใช่ master)
  2. 02Clients → Create client
    Client type: OpenID Connect · Client ID: b1dx-erp · Name: B1DX ERP (ไม่บังคับ)
  3. 03Capability config
    Client authentication: ON (confidential — จะได้ secret) · Authentication flow: เปิดแค่ Standard flow · ปิด Direct access grants / Implicit flow / Service accounts
  4. 04Login settings
    ใส่ Root URL / Redirect URIs / Web origins — ดูค่าตาม tab Dev/Prod ใน Section 3 ด้านล่าง
  5. 05Save → copy Client secret
    กด Save → ไปแท็บ Credentials → copy Client secret เก็บไว้ (ห้ามใส่ลง repo ตรง ๆ — ใส่ env/secret manager เท่านั้น)
  6. 06ตรวจ Client scope mappers
    ให้มี mapper aud = b1dx-erp (per-app audience) + claims ที่ ERP ใช้ เช่น tenant_id, roles

Env Vars ฝั่ง ERP App

หลัง save client แล้ว ใส่ค่าเหล่านี้ใน .env.local (dev) หรือ secret manager (prod) — ชื่อ env var จริงปรับตาม framework/library ที่ ERP ใช้ (เช่น NextAuth)

env · .env.local (dev)
KEYCLOAK_URL=https://keycloak.benzapps.com
KEYCLOAK_REALM=b1dx
KEYCLOAK_CLIENT_ID=b1dx-erp
KEYCLOAK_CLIENT_SECRET={secret จากแท็บ Credentials}
NEXTAUTH_URL=http://localhost:3000
🚫
ห้าม commit client secret ลง repo

ใส่ .env.local (gitignored) เท่านั้น — prod ใส่ผ่าน secret manager/CI variable

Section 05
OIDC Login Flow — Authorization Code + PKCE

ลำดับที่เกิดขึ้นจริงตอน dev กด login บนเครื่อง — front-channel (browser เห็น) สลับกับ back-channel (server คุยกันเอง ผ่าน client_secret)

sequenceDiagram
  autonumber
  actor Dev as Browser (Dev)
  participant ERP as ERP App (Next.js :3000)
  participant KC as Keycloak (realm b1dx)

  Dev->>ERP: คลิก "Login"
  ERP-->>Dev: 302 redirect → /auth (code_challenge, PKCE)
  Dev->>KC: GET /realms/b1dx/protocol/openid-connect/auth
  KC-->>Dev: แสดงหน้า Login
  Dev->>KC: กรอก username / password
  KC-->>Dev: 302 redirect → callback?code=...
  Dev->>ERP: callback พร้อม authorization code
  ERP->>KC: POST /token (code + client_secret + code_verifier)
  KC-->>ERP: access_token / id_token / refresh_token
  ERP-->>Dev: Set-Cookie session → เข้าหน้า Dashboard
💡
Realm เดียวกันทั้ง 8 ระบบ

Dev ที่ login OMS/FF ได้อยู่แล้ว ใช้ account เดิม login ERP ได้เลย — สิทธิ์การเข้าถึงแต่ละระบบยังคุมแยกที่ role/group ฝั่ง app เอง ไม่ได้ปลดล็อกอัตโนมัติ

Section 06
Troubleshooting — คลิกเพื่อดูวิธีแก้
invalid_redirect_uri พบบ่อยสุด
Redirect URI ที่ ERP ส่งไปไม่ตรงกับที่ตั้งไว้ใน Keycloak แบบ exact string (รวม trailing slash / http vs https / port) — เช็ค Clients → b1dx-erp → Login settings → Valid redirect URIs ให้ตรงกับ callback URL จริงที่ ERP ใช้
PKCE code_verifier mismatch
code_verifier ที่ส่งไปตอนแลก token ต้องเป็นตัวเดียวกับที่ใช้ generate code_challenge ตอน redirect ไป login — ปกติเกิดจาก session/cookie ที่เก็บ code_verifier หายระหว่างทาง (เช่น cookie ถูก block, หรือ redirect ข้าม domain โดยไม่ได้ตั้งใจ)
invalid_client
Client secret ผิด หรือไม่ได้ส่ง client_secret ตอนเรียก /token — b1dx-erp เป็น confidential client จึงต้องส่ง secret เสมอ (ต่างจาก public client อย่าง b1dx-portal ที่ไม่มี secret)
CORS error ตอนเรียกหน้า Keycloak
Origin ของ ERP ไม่ได้อยู่ใน Web origins ของ client — เพิ่ม origin จริง (เช่น http://localhost:3000) หรือใช้ + เพื่อ auto-match กับ redirect URIs ที่ตั้งไว้
Login สำเร็จแต่ ERP validate token ไม่ผ่าน
เช็ค issuer URL ที่ ERP ใช้ validate ต้องตรงกับ https://keycloak.benzapps.com/realms/b1dx เป๊ะ ๆ (ผิด scheme/trailing slash ก็ fail) และเช็คว่า clock บนเครื่อง dev sync ตรงเวลาจริง (JWT exp/iat sensitive กับ clock skew)
Login สำเร็จแต่ user ไม่มีสิทธิ์ใช้ฟีเจอร์ใน ERP
Login ผ่าน realm แปลว่า identity ถูกต้อง แต่ authorization ของ ERP ยังต้อง assign role/group ให้ user คนนั้นแยกต่างหาก — ไม่ใช่ bug ฝั่ง Keycloak
