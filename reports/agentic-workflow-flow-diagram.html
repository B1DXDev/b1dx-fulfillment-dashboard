Risk Override — Security (conditional)

ไม่ใช่ทุกงานเรียก security — แต่ถ้า classify เจอ surface เสี่ยง จะ บังคับ inject เข้า route หลักอัตโนมัติ แม้ user ไม่ขอ

  • แตะ auth · JWT · tenant · payment · PII · OAuth · file-upload
  • → inject 🛡 security:threat ก่อน build
  • → inject 🛡 security:scan ขนานกับ coders
  • → inject 🛡 security:pentest หลัง ui-tester
  • → บังคับ reviewer (opus) + finding High = Tier 1 stop

Gates — บังคับทุก route (ไม่แตะ)

Hard Gates เดิมยังคุมทุกเส้นทาง — dynamic เปลี่ยนแค่ "เรียกใคร" ไม่เปลี่ยน "อะไรบังคับก่อน commit"

  • plan.md lifecycle: Draft → Confirmed → In Progress → Done
  • Pre-Build Plan Gate + user trigger (build / ลุย)
  • Verify gate: build · test · lint · typecheck ต้องผ่าน
  • Commit/Push boundary: agent stage ได้ แต่รอ user สั่ง
  • Auto-Proceed Stop List (13 เงื่อนไข) ยังทำงาน
ปิดลูป — feedback กลับเข้า classify
deploy→ monitor (Grafana / Sentry)→ incident / bug พบใหม่→ กลับเข้า Phase 0 classify ↻
