Normalize vs Denormalize

เก็บให้ถูกที่ source  ·  เตรียมให้เร็วที่ read layer

Write / Source

Normalize

เก็บข้อมูลแต่ละชนิดไว้ ที่เดียว ตารางอื่นอ้างอิงผ่าน Foreign Key แทนการคัดลอก

แก้ email ลูกค้า ครั้งเดียว → ถูกทุก order ทันที

เก็บให้ถูกต้อง
Read / Report

Denormalize

คำนวณ JOIN ไว้ล่วงหน้า เก็บผลลัพธ์สำเร็จรูป ดึงมาแสดงได้เลยโดยไม่ต้องคิดใหม่

รายงานที่เคยรอ 3 ชั่วโมง → 2 วินาที

ดึงให้เร็ว
คำนวณ / sync / refresh ไว้ล่วงหน้า
┌─────────────────────────────────────────────────────────┐ │ WRITE — Normalized (Source of Truth) │ │ │ │ customers ──┐ │ │ products ──┼──→ orders (FK อ้างอิงเท่านั้น) │ │ warehouses──┘ │ │ │ │ แก้ที่เดียว = ถูกทุกที่ · ไม่มีข้อมูลซ้ำ │ └────────────────────────────┬────────────────────────────┘ │ Materialized View / ETL / CDC │ คำนวณ JOIN ไว้ล่วงหน้า ↓ ┌─────────────────────────────────────────────────────────┐ │ READ — Denormalized (Pre-computed) │ │ │ │ order_summary_view │ │ ├── ชื่อลูกค้า ← รวมมาจาก customers แล้ว │ │ ├── ชื่อสินค้า ← รวมมาจาก products แล้ว │ │ └── ยอดรวม ← คำนวณแล้ว ไม่ต้อง SUM ใหม่ │ │ │ │ 1 query · ไม่มี JOIN · < 1 วินาที │ └─────────────────────────────────────────────────────────┘

ทำงานอย่างไรในทางปฏิบัติ

1
ผู้ใช้สั่งซื้อ / แก้ข้อมูล
เขียนลง Normalized DB — แต่ละตารางเก็บข้อมูลของตัวเอง ไม่ซ้ำซ้อน
Primary DB (PostgreSQL)
2
ระบบคำนวณและเตรียมข้อมูล
Materialized View / ETL job ทุกคืน / CDC real-time — JOIN ตารางและคำนวณผลลัพธ์ไว้ล่วงหน้า
อัตโนมัติ — ผู้ใช้ไม่รู้สึก
3
ผู้ใช้ดูรายงาน / Dashboard
ดึงจาก Read Layer โดยตรง — ไม่มี JOIN ไม่กระทบ Production ได้ผลทันที
Read Layer (View / Cache / Data Warehouse)
🍽️

ร้านอาหาร

Normalize: ครัวเก็บวัตถุดิบแยก
Denormalize: จัดจานพร้อมเสิร์ฟ
🏢

สำนักงาน

Normalize: แต่ละแผนกเก็บข้อมูลของตัวเอง
Denormalize: รายงาน CFO รวบมาให้แล้ว
🛒

ซูเปอร์มาร์เก็ต

Normalize: โกดังเก็บของแยกประเภท
Denormalize: จัดวางชั้นวางพร้อมขาย

จำสั้น ๆ

Write = เก็บให้ถูก
Read = เตรียมให้เร็ว
ทั้งสองทำคนละชั้น ไม่ขัดกัน

ไม่ต้องเลือกอย่างใดอย่างหนึ่ง —
Normalize ที่ต้นทาง ให้ข้อมูลถูกต้องเสมอ
Denormalize ที่ปลายทาง ให้รายงานเร็วโดยไม่กระทบระบบหลัก
