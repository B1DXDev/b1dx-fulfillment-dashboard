Objective

หลังจาก Sync Worker upsert marketplace order เข้าระบบสำเร็จแล้ว ให้ auto-resolve SKU code ของแต่ละ order item กับตาราง sku_mappings แล้วเขียนผลการ match ลงในฐานข้อมูลทันที — mirror logic เดียวกับ BE-02 (manual match) เพื่อให้ order ที่ sync เข้ามาได้รับสถานะ match โดยอัตโนมัติโดยไม่ต้อง trigger ด้วย manual action

Process Flow
1

Sync Worker Tick Trigger

BackgroundService ตื่นขึ้นตาม interval

2

ClaimUnsynced()

Lock batch ของ orders ที่ยังไม่ sync ใน DB

3

UpsertOrderAsync() Existing

Upsert orders + order_items ลงใน OMS DB (idempotent)

4

ShopCredentialCache.Resolve() New

Lookup integration_credentials → oms_shop_id
Cache per shop per sync-run · unlinked shop → unmatched + warn

5

SkuMappingRepository.GetAsync() New

Query sku_mappings · Cache ทั้งหมดไว้ในหน่วยความจำ per sync-run

6

SkuMatchService.MatchItems() New

Match แต่ละ item SKU กับ mapping → List<SkuMatchResult>

7

MatchStatusCalculator.Calculate() New

คำนวณ match_status → matched / partial / unmatched
Byte-for-byte mirror ของ BE-02 calculator

8

ApplyAutoMatchAsync() SQL Write Output

Bulk write ผลลง DB พร้อม oms_shop_id via COALESCE

Branch: oms_shop_id found?

  • YES → ดำเนินการ match ต่อ
  • NO → ตั้ง match_status = unmatched + log warning แล้วข้ามไป order ถัดไป (ไม่ crash)
